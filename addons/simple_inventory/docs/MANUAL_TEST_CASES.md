# 手动测试用例文档

## 📋 概述

本文档提供详细的手动测试用例，用于验证背包系统的各项功能。每个测试用例包含测试目的、前置条件、测试步骤和预期结果。

---

## 🎯 测试环境准备

### 前置条件详细说明

#### 1. 项目设置

**步骤1.1：启用插件**
1. 打开 Godot 编辑器
2. 进入 `项目 -> 项目设置 -> 插件`
3. 找到 `Simple Inventory` 插件
4. 确保插件已启用（复选框已勾选）

**步骤1.2：配置 AutoLoad 单例**
1. 进入 `项目 -> 项目设置 -> AutoLoad`
2. 确认以下单例已配置：
   - `ItemManager` -> `res://addons/simple_inventory/scripts/managers/item_manager.gd`
   - `ItemUseStrategyManager` -> `res://addons/simple_inventory/scripts/managers/item_use_strategy_manager.gd`
3. 如果未配置，点击"添加"按钮添加这些单例

**验证方法**：
- 运行游戏，在控制台不应看到单例初始化错误
- 可以在脚本中使用 `ItemManager` 和 `ItemUseStrategyManager` 全局变量

#### 4. 初始化流程说明

**系统启动时的初始化顺序**：

1. **AutoLoad 单例初始化**
   - `ItemManager._ready()` 被调用
   - 自动加载基础资源（物品类型、装备类型）

2. **主场景初始化**
   - `main.gd._ready()` 被调用
   - 调用 `_initialize_test_resources()` 创建测试资源
   - 调用 `inventory_panel.initialize(player)` 初始化UI

3. **玩家实体初始化**
   - `player.tscn` 场景加载
   - `InventoryComponent._ready()` 初始化背包
   - `EquipmentComponent._ready()` 初始化装备系统

**验证初始化是否成功**：

运行游戏后，在控制台应该看到：
```
测试资源初始化完成
已创建并注册所有测试物品配置
```

如果没有看到这些消息，检查：
- `ItemManager` 是否正确配置为 AutoLoad
- `main.gd` 是否正确附加到主场景
- 是否有错误信息输出

#### 5. 测试命令说明

**可用命令**：

1. `add_item <item_id> <quantity>`
   - 示例：`add_item potion_health 5`
   - 功能：添加指定数量的物品到背包

2. `remove_item <slot_index>`
   - 示例：`remove_item 0`
   - 功能：移除指定槽位的物品

3. `remove_item_by_id <item_id> <quantity>`
   - 示例：`remove_item_by_id potion_health 3`
   - 功能：通过物品ID移除指定数量的物品

4. `pack_inventory`
   - 示例：`pack_inventory`
   - 功能：整理背包（合并相同物品、按类型排序）

5. `list_inventory`
   - 示例：`list_inventory`
   - 功能：查看背包中所有物品的状态

6. `get_item_count <item_id>`
   - 示例：`get_item_count potion_health`
   - 功能：查看指定物品的数量

7. `equip_item <item_id>`
   - 示例：`equip_item sword_1`
   - 功能：装备指定ID的物品（从背包中查找并装备）

8. `unequip_item <slot_id>`
   - 示例：`unequip_item weapon_slot`
   - 功能：卸载指定槽位的装备

9. `list_equipment`
   - 示例：`list_equipment`
   - 功能：查看当前装备状态

**命令输入方式**：
- 在游戏中的 `LineEdit` 输入框输入命令
- 按回车或点击提交按钮执行

---

#### 6. 常见问题排查

**问题1：物品无法添加**
- 检查 `ItemManager` 是否正确初始化
- 检查物品是否已注册（查看控制台输出）
- 检查物品ID是否正确

**问题2：装备无法装备**
- 检查装备类型是否已注册
- 检查装备槽位配置是否正确
- 检查 `EquipmentComponent` 是否正确初始化

**问题3：UI不显示**
- 检查 `InventoryPanel` 是否正确初始化
- 检查玩家实体是否正确传递
- 检查UI场景是否正确加载

**问题4：控制台报错**
- 查看错误信息，定位问题
- 检查资源路径是否正确
- 检查类名是否正确（GameplayItem、GameplayEquip等）
